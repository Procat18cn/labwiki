## 维护前准备

### 1. 确保环境就绪

```bash
# 进入项目目录
cd labwiki

# 激活conda环境
conda activate mydocs

# 验证MkDocs版本
mkdocs --version
# 应显示: mkdocs, version 1.6.x
```

### 2. 更新本地代码（多人协作时）

```bash
# 拉取最新代码
git pull origin main

# 如有冲突，先解决冲突再继续
```

### 3. 启动本地预览

```bash
# 启动开发服务器
mkdocs serve

# 浏览器访问 http://127.0.0.1:8000
```

> 保持 `mkdocs serve` 在后台运行，Markdown 内容修改通常会实时刷新。若修改 `extra.css`、`extra.js`、`mkdocs.yml` 或 `overrides/` 后浏览器仍显示旧样式/旧结构，请重启 `mkdocs serve`，并使用 `Ctrl+Shift+R` 强制刷新页面。

## 更新知识文章

### 新增文档

**步骤1：确定文档位置**

根据内容分类，选择对应目录：

| 分类 | 目录路径 | 内容示例 |
|------|----------|----------|
| 实验环境与系统 | `docs/environments/` | Ubuntu/Windows配置、环境搭建 |
| 软件开发与工具 | `docs/development/` | Git使用、IDE配置、编程技巧 |
| AI与深度学习 | `docs/ai-deep-learning/` | PyTorch、模型训练、应用案例 |
| 实验技能与方法 | `docs/lab-skills/` | 实验操作、数据分析、测试方法 |
| 实验系统与硬件 | `docs/hardware/` | 设备配置、仪器使用 |
| 附录 | `docs/appendix/` | 命令速查表、索引 |

**步骤2：创建文档文件**

```bash
# 示例：创建Ubuntu网络配置文档
touch docs/environments/ubuntu/network-config.md
```

**步骤3：编写文档内容**

使用标准文档模板：

```markdown
# 文档标题

## 摘要

简要描述本文档的内容和用途（50-100字）。

## 关键词

关键词1, 关键词2, 关键词3

## 正文

### 第一节标题

详细内容...

### 第二节标题

详细内容...

## 参考文档

- [相关链接1](url)
- [相关链接2](url)

---

*更新日期: 2026-04-18*
```

**步骤4：更新导航配置**

编辑 `mkdocs.yml`，在 `nav` 部分添加新文档：

```yaml
nav:
  - 实验环境与系统:
    - environments/index.md
    - Ubuntu系统:
      - environments/ubuntu/system-basics.md
      - environments/ubuntu/user-permissions.md
      - environments/ubuntu/network-config.md  # <-- 新增
```

**步骤5：验证显示**

- 检查左侧导航是否正确显示新文档
- 检查文档内容渲染是否正常
- 检查内部链接是否有效

### 修改现有文档

```bash
# 直接编辑现有文件
# 例如：修改Ubuntu基础文档
vim docs/environments/ubuntu/system-basics.md
```

修改后自动刷新，验证内容即可。

### 删除文档

**步骤1：删除文件**

```bash
rm docs/environments/ubuntu/old-document.md
```

**步骤2：更新导航配置**

编辑 `mkdocs.yml`，移除对应的导航项。

**步骤3：检查是否有其他文档引用该文件**

```bash
# 搜索引用
grep -r "old-document.md" docs/
```

如有引用，需同步更新或删除。

## 更新主页内容

主页文件位于 `docs/index.md`。

### 修改主页结构

主页通常包含以下部分：

```markdown
# 实验室知识库

## 简介

知识库的简要介绍...

## 知识分类

- [实验环境与系统](environments/)
- [软件开发与工具](development/)
- ...

## 快速开始

...使用说明...

## 最近更新

- 2026-04-18: 新增XXX文档
- 2026-04-17: 更新YYY内容

## 维护信息

- 维护团队: XXX实验室
- 联系方式: xxx@example.com
```

### 使用图标增强可读性

MkDocs Material 支持 Material Design 图标：

```markdown
- :material-desktop-tower: **实验环境与系统** - 系统配置与环境搭建
- :material-code-braces: **软件开发与工具** - 开发工具与编程技巧
- :material-brain: **AI与深度学习** - 人工智能与深度学习
```

### 添加最近更新记录

建议在主页维护一个"最近更新"区块，帮助用户了解内容变化。

## 更新目录结构

### 添加新分类

**步骤1：创建目录和索引文件**

```bash
# 创建新分类目录
mkdir docs/new-category

# 创建索引文件
touch docs/new-category/index.md
```

**步骤2：编写分类索引页**

`docs/new-category/index.md`:

```markdown
# 新分类名称

## 概述

该分类的内容概述...

## 文档列表

- [文档1](doc1.md)
- [文档2](doc2.md)
```

**步骤3：更新导航配置**

```yaml
nav:
  - 首页: index.md
  - 实验环境与系统:
    - environments/index.md
    - ...
  - 新分类名称:          # <-- 新增
    - new-category/index.md
```

**步骤4：更新主页分类链接**

在 `docs/index.md` 中添加新分类的入口。

### 重组目录结构

如需大规模重组目录：

1. **规划新结构** - 在纸上画出目标目录树
2. **批量移动文件** - 使用 `git mv` 保留历史记录
3. **批量更新链接** - 使用编辑器全局替换
4. **更新导航配置** - 同步修改 `mkdocs.yml`
5. **全面验证** - 检查所有页面是否正常

```bash
# 示例：批量移动文件
git mv docs/old-path/file.md docs/new-path/file.md
```

## 本地预览与验证

### 基础验证清单

每次修改后，检查以下项目：

- [ ] 页面能正常加载，无404错误
- [ ] 导航栏正确显示当前位置
- [ ] 文档内容渲染正常（标题、列表、代码块等）
- [ ] 内部链接可正常点击跳转
- [ ] 搜索功能能搜索到新内容
- [ ] 深色/浅色模式切换正常

> 调试提示：`mkdocs serve` 对普通 Markdown 内容的热刷新较可靠；对 `extra.css`、`extra.js`、`mkdocs.yml`、`overrides/` 等站点样式、脚本和主题覆盖文件，预览服务有时会继续返回旧资源。若页面效果与源码不一致，应重启 `mkdocs serve`，并在浏览器中强制刷新。

### 搜索功能验证

```bash
# 在搜索框输入关键词，验证：
# 1. 搜索结果出现
# 2. 结果数量合理
# 3. 点击结果能正确跳转
```

### 移动端验证（可选）

```bash
# 使用浏览器开发者工具
# 1. 按 F12 打开开发者工具
# 2. 点击设备模拟按钮
# 3. 选择手机尺寸测试响应式布局
```

## 提交与部署

### 提交规范

```bash
# 1. 查看修改的文件
git status

# 2. 添加修改的文件
git add docs/environments/ubuntu/new-doc.md

# 3. 提交（使用清晰的提交信息）
git commit -m "新增: Ubuntu网络配置文档

- 添加网络配置基础内容
- 包含静态IP和DHCP配置方法
- 更新导航配置"

# 4. 推送到GitHub
git push origin main
```

### 提交信息规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 新增文档 | `新增: 文档标题` | `新增: Ubuntu防火墙配置指南` |
| 修改文档 | `更新: 文档标题 - 修改内容` | `更新: Git使用指南 - 添加rebase说明` |
| 删除文档 | `删除: 文档标题 - 原因` | `删除: 旧版安装指南 - 已过时` |
| 修复错误 | `修复: 问题描述` | `修复: 修复broken链接` |

### 验证部署

推送后：

1. 访问 GitHub 仓库的 **Actions** 标签页
2. 确认部署工作流成功完成（绿色勾）
3. 访问线上网站验证更新

## 安全注意事项

### 1. 不要提交敏感信息

❌ **禁止**在文档中包含：
- 密码、API密钥、Token
- 服务器IP地址（内网）
- 个人隐私信息

✅ **应该**：
- 使用占位符如 `<YOUR_API_KEY>`
- 将敏感配置放在环境变量中
- 使用 `.gitignore` 排除敏感文件

### 2. 图片和资源管理

- 图片统一放在 `docs/assets/images/` 目录
- 使用相对路径引用：`
- 大文件（>10MB）考虑使用外部图床

### 3. 定期备份

GitHub 本身即是备份，但建议：
- 定期 `git push` 推送修改
- 重要修改前先创建分支

```bash
# 创建功能分支（大型修改时）
git checkout -b feature/new-section

# 完成后再合并到main
git checkout main
git merge feature/new-section
```

### 4. 避免破坏现有链接

移动或重命名文件时：
- 使用 `git mv` 而非直接删除/创建
- 检查是否有其他文档引用该文件
- 必要时添加重定向（MkDocs不支持原生重定向，需手动更新链接）
