## 描述

在实验室的运行、维护、软硬件开发、专业学术研究过程中，会产生大量且涉及多方面的知识、经验、技巧等（以下统称为“知识”）。本知识库用于将这些知识进行集中管理、维护和查阅。

- **知识库组织形式：** 将结构化的知识文档（`.md`文件）进行集中管理（位于`docs`文件夹）。文件夹内以 *分类* 且 *多级* 的目录架构储存知识文档。该知识库提供可视化网页作为接口，用于知识文档/文章的查找、阅读、下载。知识库具有可维护的能力，可进行知识文档的修改、增减、勘误、补充等。

- **知识库网页主要架构：** 知识库采用单一页面框架，其中包括 - 左边栏 & 主窗口。左边栏的内容由上到下分别为 - 知识库logo/标题、搜索栏、（多级）目录索引。主窗口用于显示主页和知识文档正文。主页是初次进入该知识库时显示的页面，其中显示知识库标题、简介、主要内容索引、开发/维护/联系信息等内容（可以参考示例`Temp_Examples\预览-实验室知识库.html`）。通过左边栏索引、搜索、或主页索引，主窗口切换进入具体知识文档正文页，该页面与结构化的知识文档（`.md`文件）连接并将其可视化。在正文页面右侧位置，还提供当前页面的辅助目录导航和下载按钮。下载按钮可以将当前知识文档保存/下载到本地。

- **知识库网页风格设计：** 以科学研究、AI、深度学习、现代化为美术风格关键词。在满足基本功能要求的前提下，适当减少其他花哨显示内容（比如华丽的动画、动态效果等）。页面、字体、文字颜色必须保证页面内容的高可读性。其他细节包括：左边栏索引的各级子目录可折叠；左边栏可向左收缩隐藏和调出。

- **部署：** 该网页可以在本地部署（需依托本地conda环境`mydocs`），也可以托管到GitHub平台并随时随地打开，可供团队小伙伴们进行知识的共享。

- **本描述时间戳-版本：** 2026.04.18 - v1 （更新）

**开发日志 (v5)：**

- **问题1修复 - 调色板按钮明亮模式可见性：** Material 主题默认 header 为深色背景，其 `.md-header__button` 图标为白色。由于本项目顶栏为浅色背景，导致图标与背景融为一体。修复方案：为 `.lab-header__palette .md-header__button` 显式设置 `color: var(--md-default-fg-color--light) !important`（灰色），hover 时变为主题色。修改文件：`overrides/extra.css`。
- **问题2修复 - 搜索框居中对齐：** 根因为顶栏 flex 布局中左右宽度不对称导致搜索框偏右。初次修复采用 `flex: 0 1 36rem` 方案，但引发搜索功能失效（见下方"重大修复"）。最终方案：将顶栏 `.md-header__inner` 从 Flexbox 改为 **CSS Grid 三栏布局** (`grid-template-columns: 1fr minmax(0, 36rem) 1fr`)，左右两栏等宽自动平衡，中间栏固定最大宽度，实现搜索框真正居中。修改文件：`overrides/main.html`（添加 `.lab-header__left` 包裹容器）、`overrides/extra.css`。
- **问题3修复 - 点击外部/Escape关闭搜索框：** Material 主题搜索通过 `#__search` checkbox 控制开关状态。添加 `initSearchDismiss()` 函数，包含三部分：(1) 聚焦搜索输入框时自动勾选 `#__search` 以激活搜索结果面板；(2) 监听 `mousedown` 事件（比 `click` 更可靠），当搜索激活且点击目标不在 `.md-search` 容器内时，取消勾选并清空输入框；(3) 监听 `keydown` 事件，Escape 键关闭搜索并清空输入。修改文件：`overrides/extra.js`。
- **重大修复 - 搜索功能失效：** 初次问题2修复后，搜索框样式无变化且搜索功能失效。根因分析：Material 主题在移动端视口（<960px）通过 CSS 将 `.md-search__inner` 设为 `height: 0; opacity: 0; width: 0; overflow: hidden`，而桌面端（>=960px）使用 `float: right; width: 11.7rem` 配合 checkbox 展开机制。自定义 flex 修改破坏了此行为链。修复方案：(1) CSS 中使用 `!important` 强制覆盖 `.md-search__inner` 的 width/height/opacity/overflow/position/transform/float，使搜索栏始终可见；(2) 添加 `.md-search__output` 绝对定位使搜索结果正确下拉展示；(3) 添加 `.md-search__overlay` 样式，搜索激活时显示半透明遮罩层。修改文件：`overrides/extra.css`、`overrides/extra.js`。
- **其他：** 按要求已终止后台运行的 mkdocs 服务。预览前需手动启动 `mkdocs serve`。

**开发日志 (v4)：**

- **问题1修复 - 合并双搜索框：** 移除了自定义的 `lab-header__search` 触发标签，直接使用 Material 主题原生 `.md-search` 组件作为顶栏居中搜索栏。通过 CSS 设置 `.md-search { flex: 1; max-width: 36rem; margin: 0 auto; }` 实现居中布局，并为搜索输入框添加边框和圆角样式。修改文件：`overrides/main.html`、`overrides/extra.css`。
- **问题2修复 - 调色板切换按钮不可见：** 根因分析发现，自定义 `{% block header %}` 覆盖了 Material 默认的 `partials/header.html`，导致 `partials/javascripts/palette.html` 初始化脚本丢失。该脚本负责从 localStorage 读取用户偏好并设置 `data-md-color-scheme` 属性。修复方案：在 header 标签内补充该初始化脚本的 include，同时添加 CSS 回退规则确保标签可见。修改文件：`overrides/main.html`、`overrides/extra.css`。
- **问题3修复 - 文档下载功能：** 创建 MkDocs hook (`hooks/copy_sources.py`)，在构建时将 `docs/` 中所有 `.md` 文件复制到 `site/_sources/` 目录。在顶栏右侧（调色板切换按钮旁）添加下载按钮，使用 HTML download 属性触发浏览器下载。通过 `page.file.src_path` 动态生成下载链接，并用 Jinja2 replace 过滤器解决 Windows 路径分隔符问题。404 页面通过条件判断跳过下载按钮渲染。新增文件：`hooks/copy_sources.py`。修改文件：`overrides/main.html`、`overrides/extra.css`、`mkdocs.yml`（添加 hooks 配置）。
