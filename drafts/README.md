# 草稿区

本目录用于存放准备进入知识库、但仍需审阅和修改的文章草稿。

草稿区不参与 MkDocs 站点构建，也不会被当前文档下载 hook 复制到 `site/_sources/`。因此，使用 `organize-tech-knowledge-doc` skill 整理零散技术材料时，初稿应优先放在这里，确认后再移动到 `docs/` 对应分类目录。

注意：如果本仓库是公开仓库，提交到本目录的内容仍然是公开可见的。不要在这里保存密码、Token、API key、内网地址、个人隐私、未经清理的聊天记录或其他敏感信息。

草稿转入 `docs/` 前，请检查：

- 内容适合公开发布。
- 事实、命令、链接和引用来源已经核查。
- 文章结构符合知识库规范：标题、摘要、关键词、结构化正文、参考文档和更新日期。
- 涉及系统级操作时包含风险说明、验证方法和回滚步骤。
- 已同步更新 `mkdocs.yml`、首页入口、分类索引和必要的附录索引。
